# CLAUDE.md тАФ Morning Brief Bot

A personal Telegram bot that sends Abay one carefully-shaped message every morning at 7:00 Warsaw time. Replaces doom-scrolling. Built and maintained by one person.

## What it does

Each morning the bot generates and sends a Telegram message with six sections:

1. Date header
2. Today's person тАФ historical/scientific/philosophical figure with a takeaway
3. Principle of the day тАФ durable mental model (Stoicism, Munger, Taleb, etc.)
4. Three things in tech тАФ picked from HN + RSS, each with a "what it is" + "for you" hook
5. Concept of the day тАФ the day's AI engineering curriculum topic
6. Yesterday's check тАФ a question on yesterday's concept

The message is generated by calling Claude API with a synthesis prompt (see `prompts/synthesize.md`) given today's pre-fetched inputs.

## Architecture

```
GitHub Actions cron (0 5 * * *  тЖТ  7:00 Warsaw, summer time)
        тЖУ
Python script runs in the action
        тЖУ
1. Read state.json тЖТ current day, recent picks, yesterday's topic
2. Pick today's person, principle, concept (avoiding recent ones)
3. Fetch news: HN top 30 + RSS feeds (parallel)
4. Build JSON payload, render synthesize.md prompt
5. Call Anthropic API тЖТ markdown morning brief
6. Send to Telegram via Bot API
7. Update state.json тЖТ commit & push back to repo
```

No servers. No databases. State lives in the repo.

## State schema

`state.json` (committed to repo, updated each run):

```json
{
  "current_day": 1,
  "recent_people": ["Richard Feynman"],
  "recent_principles": ["Antifragility"],
  "yesterday_topic": null,
  "last_run": "2026-04-27T07:00:00+02:00",
  "replies": []
}
```

- `current_day` increments on every successful send (1тАУ120, then loop or stop)
- `recent_people` keeps last 60 used (cycle target: ~60 days)
- `recent_principles` keeps last 30 used (cycle target: ~30 days тАФ repetition is good for tools)
- `yesterday_topic` is the previous day's curriculum topic, used for the "yesterday's check" section
- `replies` accumulates user replies to "yesterday's check" тАФ for v2 (round-trip feedback)

## Selection logic

- **Person:** pick uniformly from `people.yaml` excluding `recent_people`. Aim for tradition variety тАФ don't pick a scientist if yesterday was a scientist.
- **Principle:** pick uniformly from `principles.yaml` excluding `recent_principles`. Tradition variety same rule.
- **Concept:** `curriculum.yaml[current_day - 1]`. Direct lookup, no random.

## Build order (recommended)

1. **Repo skeleton** тАФ directories, `requirements.txt` (pyyaml, anthropic, requests, feedparser).
2. **state.py** тАФ `load_state()` / `save_state()` against `state.json`. Test with a sample state.
3. **selectors.py** тАФ `pick_person(state, people)`, `pick_principle(state, principles)`, `get_concept(state, curriculum)`. Pure functions, easy to unit-test.
4. **news.py** тАФ fetch HN top 30 via `https://hacker-news.firebaseio.com/v0/topstories.json`, fetch ~2 RSS feeds (TLDR AI, The Sequence тАФ pick what's easy). Return list of `{title, summary, url, source}`. Time-box to 10 seconds total.
5. **synthesizer.py** тАФ read `prompts/synthesize.md`, replace `{{json_payload}}` with the actual payload, call Anthropic API, return string.
6. **telegram.py** тАФ send message via Bot API. Handle markdown formatting. Telegram bot tokens come from `@BotFather`; chat_id is the user's own Telegram user ID (get it by messaging the bot then hitting `getUpdates`).
7. **main.py** тАФ wire it all together. Read state, fetch, synthesize, send, update state.
8. **Dry-run mode** тАФ `python -m src.main --dry-run` should print the brief instead of sending. Critical for testing without spamming yourself.
9. **GitHub Actions workflow** тАФ `.github/workflows/daily.yml`. Cron at `0 5 * * *` (UTC; equals 7:00 Warsaw during summer time, 6:00 during winter). Permissions: `contents: write` so it can commit `state.json`. Three secrets: `ANTHROPIC_API_KEY`, `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`.
10. **Manual test** тАФ `workflow_dispatch` in the workflow lets you trigger from the GitHub UI without waiting for cron.

## Telegram bot setup (one-time)

1. Message `@BotFather` on Telegram. `/newbot`. Pick a name. Get the bot token.
2. Message your new bot once (any message).
3. Call `https://api.telegram.org/bot<TOKEN>/getUpdates` in a browser. Find `message.chat.id` in the JSON.
4. Save token + chat_id as GitHub secrets.

## Time zone math

- GitHub Actions cron is UTC.
- Warsaw is UTC+1 (winter) or UTC+2 (summer/CEST).
- For 7:00 Warsaw delivery year-round, use `0 5 * * *` (UTC) and accept that during winter it'll arrive at 6:00 Warsaw тАФ slightly early is fine, slightly late is bad.
- Or schedule at `0 6 * * *` and accept summer arrival at 8:00. Pick one.

## Tone тАФ non-negotiable

The synthesis prompt (`prompts/synthesize.md`) contains hard tone rules. The brief must NOT sound like generic LLM output. Banned phrases, no emoji, no peppy opener. Read those rules before changing anything in the prompt.

## What's deliberately NOT in v1

- Round-trip replies (user replies to the brief тЖТ bot critiques their answer). Phase 2.
- Spaced repetition for concepts (yesterday-only is enough signal initially).
- Multiple LLM judges / self-consistency on the synthesis (one call is fine).
- Fancy database for state (JSON file in repo is honest at this scale).
- Web UI (it's a personal tool тАФ Telegram IS the UI).

## Cost estimate

- Anthropic API: one Claude Opus call per day, ~10k input tokens, ~2k output tokens. Roughly $0.20/day = $6/month.
- GitHub Actions: free for personal use within limits.
- Telegram Bot API: free.
- Total: ~$6/month.

## Success criteria for v1

- Bot sends a message at ~7am Warsaw every day for 14 consecutive days without breaking.
- Brief reads more interesting than Twitter on the first 14 mornings.
- Tone matches the v3 mockup тАФ no "let's dive in", no emoji, no peppy opener.
- State commits cleanly each day (no manual intervention).
